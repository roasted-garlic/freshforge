# Plans

This directory holds **plan documents** created during the Plan phase.

## Naming

- Feature work: `YYYY-MM-DD-short-description-plan.md`
- Workflows: `project-intake-plan.md`, `new-project-bootstrap-plan.md`

## Template

Copy from: `.cursor/workflow/plan-template.md`

## Rules

- A plan must exist before implementation begins
- Plans include scope, impacts, risks, test strategy, and human checkpoints
- Blocked reviews require plan revision — create updated plan or amend with review reference

## Workflow

Plan → Review (in `docs/reviews/`) → Implement → Test → Signoff

See `docs/WORKFLOWS.md` and `.cursor/workflow/state.md`.
