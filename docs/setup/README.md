# Setup

This directory is reserved for **project setup notes** that are not universal workflow docs.

## Suggested Use

After intake or bootstrap, you may add:

- `local-dev.md` — machine-specific setup tips
- `onboarding.md` — new developer checklist
- `tooling.md` — IDE extensions, CLI tools
- Provider-specific quickstarts that supplement `BACKEND.md`

## Not Required for Starter

The workflow starter does not populate this folder by default. Add files when the team needs setup documentation beyond `BACKEND.md`, `DEPLOYMENT.md`, and `TESTING.md`.

## Agent Guidance

Agents should not invent setup scripts or install packages unless part of an approved implementation plan. Document commands here after the project's stack is known.
